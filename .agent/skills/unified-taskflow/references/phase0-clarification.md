# Phase 0: Clarification

进入 Spec-Driven 模式前的必经阶段。

## 核心规则

1. **禁止**立即创建文档或代码
2. **必须提问**：至少 3 个澄清性问题
3. **必须总结**：口头总结用户意图
4. **获得确认**后才能进入 Phase 1

## 问题框架

### 边界问题
- 这个任务包含哪些功能？
- 明确不需要做的是什么？
- 与现有系统如何集成？

### 约束问题
- 有技术限制吗？（框架、语言、依赖）
- 有时间限制吗？
- 有哪些不可变的现有代码？

### 优先级问题
- 哪个功能最重要？
- 如果时间不够，可以先放弃什么？

### 风险问题
- 有什么可能出问题？
- 需要备选方案吗？

## 输出

确认理解后，口头总结：

```
"确认一下我的理解：
1. 目标是 [xxx]
2. 包含 [A, B, C]，不包含 [D, E]
3. 约束是 [xxx]
4. 最重要的是 [xxx]

理解正确吗？"
```

用户确认后进入 Phase 1。
