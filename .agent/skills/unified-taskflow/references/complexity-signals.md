# Complexity Signals 复杂度信号

用于判断任务应使用哪个档位。

## 信号列表（每项 +1 分）

| 信号 | 示例 |
|------|------|
| 任务描述超过 50 字 | 长段落描述需求 |
| 涉及代码库/项目名称 | "重构 X"、"升级 Y 模块" |
| 包含变更动词 | 重构、实现、设计、升级、迁移 |
| 隐含多组件变更 | 前端 + 后端, API + 数据库 |
| 涉及架构/接口/数据模型 | 改 API、新增数据表 |
| 用户表达不确定性 | "不确定怎么做"、"帮我想想" |
| 涉及新技术/框架引入 | "引入 Redis"、"切换到 TS" |

## 阈值判定

```
≥ 1 分: 考虑 Standard
≥ 2 分: 建议 Standard，提示用户
≥ 3 分: 强烈建议 Spec-Driven，主动询问
```

## 兜底规则

无法判断时，默认进入 Clarification Phase 询问用户。
**宁可多问一句，不可盲目开工。**
