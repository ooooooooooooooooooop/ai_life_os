# 交互设计原则

## 选项设计

1. **优先选择题**：每次交互尽量使用选择题模式
2. **3-4 个选项为佳**：足够覆盖主要方向
3. **上下文感知**：选项根据当前 Phase 动态调整
4. **支持组合**："1 和 3"、"先 1 后 2"
5. **始终保留"其他想法"**：允许用户跳出预设

## 确认机制

- **信任等级**：low/medium/high 决定确认频率
- **批准范围**：用户可声明"技术决策自动批准"
- **延迟确认**：一组相关步骤完成后再确认

## 信息密度

| 等级 | 场景 | 内容 |
|------|------|------|
| 极简 | 进度更新 | 一句话 |
| 标准 | 阶段转换 | 总结 + 下一步 |
| 详细 | 重要决策 | 分析 + 选项 + 建议 |

## 推荐选项

- 每个选择点应有**推荐选项**
- 推荐基于：上下文、用户历史、最佳实践
- 格式：选项后加 `(推荐)` 标记

## 异常处理

| 情况 | 处理 |
|------|------|
| 无关回复 | 询问"这是回答上一个问题吗？" |
| 过于简短 | 请求澄清 |
| 矛盾信息 | 指出矛盾，请求解决 |
