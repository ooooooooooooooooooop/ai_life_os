# 中文输出模板（四视角对话制）

## 生成守护（内部自检，不要原样输出）
- 固定 7 个章节，且顺序不变。
- 固定 4 个角色组成（2 个现实人物 + 2 个抽象角色）。
- 每个对话轮次必须有 4 个角色发言。
- 未在当前上下文提供来源时，不要给现实人物添加直接引号语录。

## 1. Good Group To Explore X（四视角人群组合）
- `决策框架`：问题 + 约束 + 成功标准 + 时间范围。
- `现实人物A`：姓名 + 相关身份 + 选择理由 + 2-3 个公开证据锚点。
- `现实人物B`：姓名 + 相关身份 + 选择理由 + 2-3 个公开证据锚点。
- `领域专家抽象`：该角色代表的专业立场与决策压力。
- `全知智能体抽象`：该角色的观察范围与推理职责。
- `人选评分`：A 评分、B 评分、组合多样性评分、置信度。

说明：以上四个角色必须固定存在，不可替换为泛化面板。

## 2. Dialogue Round 1: Initial Positions（首轮立场）
- `[现实人物A]` 这类人会先说：...
- `[现实人物B]` 这类人会先说：...
- `[领域专家抽象]` 这类人会先说：...
- `[全知智能体抽象]` 这类人会先说：...

## 3. Dialogue Round 2: Cross-Examination（交叉质询）
- `[现实人物A -> 其他角色]` 主要挑战：...
- `[现实人物B -> 其他角色]` 主要挑战：...
- `[领域专家抽象 -> 其他角色]` 主要挑战：...
- `[全知智能体抽象 -> 其他角色]` 主要挑战：...
说明：每个角色至少挑战一个具体假设或关键权衡。

## 4. Dialogue Round 3: Revised Positions（修正后立场）
- `[现实人物A]` 在回应质询后的修正 + 与第一轮相比改变了什么：...
- `[现实人物B]` 在回应质询后的修正 + 与第一轮相比改变了什么：...
- `[领域专家抽象]` 在回应质询后的修正 + 与第一轮相比改变了什么：...
- `[全知智能体抽象]` 在回应质询后的修正 + 与第一轮相比改变了什么：...

## 5. Dialogue Round 4: Final Statements（最终陈述）
- `[现实人物A]` 最终主张 + 一个具体动作（负责人/时间）：...
- `[现实人物B]` 最终主张 + 一个具体动作（负责人/时间）：...
- `[领域专家抽象]` 最终主张 + 一个具体动作（负责人/时间）：...
- `[全知智能体抽象]` 最终主张 + 一个具体动作（负责人/时间）：...

## 6. Moderator Synthesis（综合结论）
- `最终建议`：...
- `为何优于最强备选`：...
- `成功前提`：...
- `预警指标`：...
- `下一步1-3个动作`：...
- `运行元数据（行内）`：使用的 profile + 未解决风险数量。

## 7. Uncertainty Ledger（不确定性台账）
- `事实`：...
- `假设`：...
- `推测`：...
- `下一步需补证据`：...

## 精简版

用户要求简短输出时：
1. 仍保留7个主标题，不可删除。
2. 每个标题控制在1-3条。
3. 四个角色每轮都要发言，可压缩为一句。
4. 仍需保留“观点为基于公开信息的模拟推断”这一安全标记。
